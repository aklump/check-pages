Add Runner Functionality

Another way to extend Check Pages is to use the with_extras() function. This method is easier and faster than writing a plugin. It is a simple means to share runner customisation across different runners, or even projects. This is the basis for the authentication providers shipped with Check Pages.

  1. Create a directory extras, for example inside your tests directory. (Note: It must be resolvable by \AKlump\CheckPages\Parts\Runner::resolve().)

  2. In that directory create a file, whose filename will be used as the first argument to with_extras(). So for with_extras('foo') you should create extras/foo.php.

    # file: runner.php

with_extras('foo', [
  "title" => "Lorem",
  "color" => "blue",
]);
...

  3. The second argument passed to with_extras() is available in extras/foo.php as $config.

    # file: extras/foo.php

$title = $config['title'];
$color = $config['color'];
  4. Also, you have access to the \AKlump\CheckPages\Parts\Runner instance in the variable $runner.
  5. You may use any of the runner_functions.inc as you might otherwise do in a runner file.

Output

You may write output if desired, for example:

echo sprintf('Base URL is %s', config_get('base_url')) . PHP_EOL;
echo \AKlump\LoftLib\Bash\Color::wrap('blue', 'foo');
