Testing Response Headers

You can assert against response headers like this:

-
  visit: /foo
  find:
    -
      header: content-type
      contains: text/html
  • header is NOT case-sensitive.

  • But contains is, so... if you're trying to match a header value with case-insensitivity, you should use the match key, with the i flag like this:

    - header: content-type
  matches: /text\/html/i

Example Tests

-
  visit: /test_subject.php
  find:
    -
      header: foo
      set: fooHeader
    -
      header: foo
      is not: pha
    -
      header: foo
      contains: pha
    -
      header: foo
      is: alpha
    -
      header: bar
      is: BRAVO
    -
      header: bar
      matches: /^bravo$/i
    -
      header: bar
      not matches: /^charlie$/i
    -
      why: This header should not even exist, testing that case.
      header: zulu
      not matches: /^charlie$/i
    -
      header: baz
      count: 0
    -
      header: content-type
      count: 1

Provided by the header handler.
